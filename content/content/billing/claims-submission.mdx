## Overview

This protocol outlines the standard procedures for submitting insurance claims at Adapt Psychiatry. Following these procedures ensures timely reimbursement and compliance with payer requirements.

## Pre-Submission Checklist

Before submitting any claim, verify the following:

- [ ] **Patient insurance information is current**
  - Verify eligibility before each visit
  - Confirm copay/deductible status
  - Check for authorization requirements

- [ ] **Authorization obtained (if required)**
  - Document authorization number
  - Verify number of approved visits
  - Check authorization date range

- [ ] **Correct CPT codes selected**
  - Code matches service provided
  - Documentation supports code level
  - Modifier applied if necessary

- [ ] **ICD-10 diagnosis codes documented**
  - Primary diagnosis most specific
  - Secondary diagnoses included if relevant
  - Codes supported by clinical documentation

- [ ] **Provider NPI included**
  - Rendering provider NPI
  - Billing provider NPI (if different)
  - Group NPI (if applicable)

- [ ] **Place of service code correct**
  - 11 = Office
  - 02 = Telehealth
  - Confirm matches actual service location

## Submission Timeline

### Clean Claims

- **Submit within:** 24 hours of service date
- **Electronic submission:** Preferred method
- **Batch submission:** Daily by 5:00 PM

### Claims Requiring Authorization

- **Submit within:** 48 hours of receiving authorization
- **Include:** Authorization number on claim
- **Verify:** Authorization covers date of service

### Corrected Claims

- **Submit within:** 7 days of identifying error
- **Use:** Corrected claim indicator
- **Include:** Explanation of correction

## Payer-Specific Requirements

### Commercial Insurance

**General Requirements:**
- Verify eligibility before each visit
- Obtain prior authorization for specific services
- Submit electronically when possible
- Follow up within 30 days

**Major Payers:**

| Payer | Portal | Phone | Notes |
|-------|--------|-------|-------|
| BCBS WNY | Availity | 1-800-XXX-XXXX | Requires ERA enrollment |
| Independent Health | Provider Portal | 1-800-XXX-XXXX | Prior auth required for many services |
| Univera | Provider Portal | 1-800-XXX-XXXX | Check benefits online |
| MVP | Provider Portal | 1-800-XXX-XXXX | Credentialing required |

### Medicare

**Requirements:**
- Ensure Medicare enrollment is current
- Use appropriate modifiers (GT for telehealth)
- Submit within 1 year of service date
- Monitor remittance advice weekly

**Common Denial Codes:**
- CO-16: Claim/service lacks information
- CO-18: Duplicate claim/service
- CO-29: Time limit for filing expired
- CO-50: Non-covered service

### Medicaid (eMedNY)

**Requirements:**
- Verify Medicaid eligibility monthly
- Obtain PA when required
- Submit through eMedNY portal
- Follow up within 45 days

**PA Requirements:**
- Initial psychiatric evaluation (90792)
- Psychotherapy over 45 minutes
- Certain medication management services

## Denial Management

### Common Denial Reasons and Resolution

| Denial Code | Reason | Resolution |
|-------------|--------|------------|
| CO-16 | Missing/invalid information | Review claim, correct errors, resubmit |
| CO-18 | Duplicate claim | Verify claim status before resubmitting |
| CO-29 | Time limit expired | Submit with documentation, appeal if needed |
| CO-45 | Charge exceeds fee schedule | Adjust per contract, write off difference |
| CO-50 | Non-covered service | Verify benefits, appeal with medical necessity |
| CO-252 | Missing/invalid prior auth | Obtain retro auth if possible, appeal |
| PR-2 | Deductible not met | Patient responsibility, bill patient |
| PR-3 | Copay amount | Patient responsibility, collect at visit |

### Appeal Process

1. **Review denial reason thoroughly**
   - Understand why claim was denied
   - Gather supporting documentation

2. **Submit appeal within payer deadline**
   - Medicare: 120 days
   - Commercial: Varies (30-180 days)
   - Include all supporting documentation

3. **Track appeal status**
   - Document appeal submission date
   - Follow up if no response within timeframe
   - Escalate if denied again
